# Constit

**Campaign operations, simplified.**

Import your contact list, generate constituent message variants with AI, coordinate volunteers, and track outreach — all in one place.

---

## What's built

- CSV import with full **column mapping UI** (handles "First Name" + "Last Name", "Smith, John", any format)
- AI message generation — **5 variants** across formal, conversational, and urgent tones
- **Hard SMS character validation** — live counter in UI, enforced server-side, prevents Twilio double-segment charges
- Contact status tracking (pending → contacted → replied → opted_out)
- CSV export with selected message attached — works with any existing SMS/email tool
- Supabase Row Level Security — every user sees only their own data

## Setup

### 1. Clone and install

```bash
git clone https://github.com/YOUR_USERNAME/constit.git
cd constit
npm install
```

### 2. Create Supabase project

1. Go to [supabase.com](https://supabase.com) → New project
2. Open the **SQL editor**
3. Paste and run the contents of `schema.sql`
4. Copy your **Project URL** and **anon key** from Settings → API

### 3. Set environment variables

```bash
cp .env.example .env.local
```

Fill in `.env.local` with your Supabase URL, anon key, and OpenAI API key.

### 4. Run locally

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000)

---

## Deploy to Vercel

```bash
# Push to GitHub first, then:
vercel --prod
```

Add your environment variables in Vercel → Project → Settings → Environment Variables. Same three keys as `.env.local`.

## SMS character limits — why this matters

SMS carriers charge per **segment**. One segment = 160 characters. If your message is 161 characters, Twilio bills you for 2 segments — double the cost, silently.

Constit enforces this at three levels:
1. **Prompt**: explicitly instructs the AI to stay under 160
2. **Server** (`lib/ai.ts`): truncates any response that exceeds 160 before storing
3. **UI** (`MessageCard.tsx`): live character counter with a red warning if a message would cause double billing

## CSV import — how column mapping works

Real voter data is messy. Constit handles:
- `Name`, `Full Name`, `Voter Name` → single name column
- `First Name` + `Last Name` → separate columns, auto-joined
- `Smith, John` format → automatically reversed to `John Smith`
- Any phone format → stripped to digits + standard separators
- Tags as comma-separated values in one column

The import shows a **column mapping screen** so the user always confirms which column is which before any data is written.

## Positioning

Constit is a **campaign operations platform** — not an AI persuasion tool. Every message generated by the AI is reviewed and approved by a human before sending. The AI is a writing assistant, not an automation engine.

This positioning matters for:
- SMS provider compliance (Twilio, Bandwidth have strict political messaging policies)
- Press coverage — "AI persuasion tool" invites hostile framing
- Adoption — political ops teams are rightly suspicious of black-box automation
